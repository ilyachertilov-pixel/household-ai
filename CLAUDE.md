## Система задач (ГЛАВНОЕ)

### Файлы задач
Каждый агент имеет свой файл задач:
- `household-data/agents/butler-tasks.json` — задачи Дворецкого
- `household-data/agents/chef-tasks.json` — задачи Повара
- `household-data/agents/cleaner-tasks.json` — задачи Уборщицы
- `household-data/agents/supply-tasks.json` — задачи Завхоза

### Структура задачи
```json
{
  "id": "уникальный-id",
  "title": "Название задачи",
  "created": "DD.MM.YYYY",
  "due": "DD.MM.YYYY",
  "source": "user | расписание | дворецкий",
  "priority": "срочный | высокий | обычный",
  "status": "pending | in_progress | done",
  "details": "Подробности (опционально)"
}
```

### Как обрабатывать запросы от семьи

**При получении любого сообщения:**
1. Прочитай файлы задач всех агентов
2. Определи кому адресован запрос
3. Если это новая задача — добавь в `pending` нужного агента
4. Если запрос требует немедленного ответа — ответь и зафиксируй как выполненное
5. Обнови файл задач (измени статус / добавь запись)

**Примеры маршрутизации:**
- «купи молоко» → Завхоз, добавить в shopping-list-home.json + задачу в supply-tasks.json
- «что на ужин?» → Повар, проверить weekly-menu.json и ответить
- «напомни убраться» → Уборщица, добавить задачу в cleaner-tasks.json
- «что у нас кончается?» → Завхоз, прочитать inventory.json и ответить

```
• [что заканчивается]
```

### Как добавлять задачи (алгоритм)

Когда добавляешь задачу в файл агента:
1. Прочитай текущий файл задач агента
2. Сгенерируй уникальный id (формат: XXX-NNN, где XXX = 3 буквы агента)
3. Добавь задачу в массив `pending`
4. Обнови поле `updated` на текущую дату
5. Запиши файл

Когда задача выполнена:
1. Удали из `pending` или `in_progress`
2. Добавь в `completed` с полем `completed_date`
3. Обнови `updated`

### ⚠️ КРИТИЧЕСКИ ВАЖНО: правильная запись в JSON

Новая задача должна добавляться **внутрь** массива `pending`, через запятую после последнего элемента.

**ПРАВИЛЬНО:**
```json
"pending": [
  { "id": "sup-001", "title": "Задача 1", ... },
  { "id": "sup-002", "title": "Задача 2", ... }
]
```

**НЕПРАВИЛЬНО** (задача снаружи массива — JSON сломается):
```json
"pending": [
  { "id": "sup-001", "title": "Задача 1", ... }
],
  { "id": "sup-002", "title": "Задача 2", ... }
]
```

Перед записью всегда проверяй: все `{` закрыты `}`, все `[` закрыты `]`, запятые между элементами массива есть, после последнего элемента запятой нет.
